# Password Factory - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目名称
Password Factory（密码工厂）

### 1.2 项目简介
一个现代化的在线密码生成器工具，帮助用户快速生成安全、强度高的随机密码，支持多种自定义选项和密码强度评估功能。

### 1.3 项目背景
在当前网络安全环境下，使用强密码是保护个人账户安全的第一道防线。然而，许多用户仍在使用弱密码或重复使用相同密码。Password Factory 旨在提供一个简单、直观、安全的工具，帮助用户生成和管理强密码。

## 2. 产品目标

### 2.1 核心目标
- 提供快速、安全的密码生成服务
- 帮助用户提升密码安全意识
- 打造简洁、易用的用户体验
- 确保完全的客户端安全（所有操作在浏览器本地完成）

### 2.2 成功指标
- 用户访问量：月活跃用户 10,000+
- 用户参与度：平均每次访问生成密码 3 次以上
- 用户满意度：Net Promoter Score (NPS) > 50
- 页面性能：首次内容绘制 (FCP) < 1.5s

## 3. 目标用户

### 3.1 主要用户群体
- **个人用户**：需要为各种在线账户创建强密码的普通用户
- **IT 专业人员**：需要频繁生成临时密码或系统密码的技术人员
- **企业员工**：公司要求使用强密码策略的员工
- **安全意识强的用户**：关注网络安全的用户群体

### 3.2 用户画像

#### 画像 1：小白用户
- 年龄：18-45 岁
- 技术水平：初级
- 需求：快速生成一个"看起来很安全"的密码
- 痛点：不知道什么样的密码算安全

#### 画像 2：技术用户
- 年龄：25-50 岁
- 技术水平：中高级
- 需求：自定义密码规则，批量生成密码
- 痛点：现有工具功能单一，不够灵活

## 5. 功能详细说明

### 5.1 基础密码生成

#### 功能描述
用户可以通过简单的选项配置，快速生成符合要求的随机密码。

#### 交互流程
1. 用户进入网站，看到密码生成器主界面
2. 系统默认生成一个 16 位包含大小写字母+数字+符号的密码
3. 用户可以通过滑块调整密码长度（8-128）
4. 用户可以勾选/取消勾选字符类型选项：
   - ☑ 大写字母 (A-Z)
   - ☑ 小写字母 (a-z)
   - ☑ 数字 (0-9)
   - ☑ 特殊符号 (!@#$%^&*...)
5. 每次修改选项后，密码自动重新生成
6. 用户点击"重新生成"按钮可刷新密码
7. 用户点击"复制"按钮复制密码到剪贴板
8. 显示复制成功提示

#### 界面元素
- 密码显示区域（大号等宽字体，可点击复制）
- 密码长度滑块（带数字显示）
- 字符类型复选框组
- "重新生成"按钮（主按钮样式）
- "复制密码"按钮（次要按钮样式）
- 密码可见性切换（眼睛图标）

#### 技术要点
- 使用 Web Crypto API (crypto.getRandomValues) 确保随机性
- 避免使用 Math.random()（不够安全）
- 确保至少包含一种字符类型
- 密码生成必须在客户端完成，不发送到服务器

### 5.2 密码强度评估

#### 功能描述
实时分析密码强度，并以可视化方式展示给用户。

#### 评估维度
1. **长度评分**（30%）
   - 8-11 位：10 分
   - 12-15 位：20 分
   - 16-19 位：25 分
   - 20+ 位：30 分

2. **字符多样性**（40%）
   - 仅数字/仅字母：5 分
   - 字母+数字：20 分
   - 字母+数字+符号：30 分
   - 包含大小写+数字+符号：40 分

3. **复杂度**（30%）
   - 检查常见密码黑名单
   - 检查重复字符
   - 检查连续字符（123、abc）
   - 检查键盘序列（qwerty）

#### 强度等级
- **弱** (0-40 分)：红色，"密码太弱，容易被破解"
- **中** (41-60 分)：橙色，"密码强度一般，建议增强"
- **强** (61-80 分)：黄色，"密码强度良好"
- **极强** (81-100 分)：绿色，"密码强度极强"

#### 界面元素
- 强度进度条（颜色渐变）
- 强度文字标签
- 改进建议列表（如："建议增加特殊字符"）

### 5.3 高级自定义选项

#### 功能描述
为高级用户提供更细粒度的密码生成控制。

#### 选项说明
1. **排除易混淆字符**
   - 排除：0 O o, 1 l I i L, 2 Z z
   - 提高密码可读性

2. **排除特定字符**
   - 用户输入要排除的字符
   - 例如：排除 `|` 和 `\` 避免命令行问题

3. **包含特定字符**
   - 强制包含用户指定的字符或字符串
   - 例如：包含 `@` 符号

4. **开头/结尾规则**
   - 必须以字母开头
   - 不能以符号结尾
   - 自定义开头/结尾字符

### 5.4 批量生成

#### 功能描述
一次性生成多个密码，适合需要为多个账户创建密码的场景。

#### 交互流程
1. 用户点击"批量生成"按钮
2. 弹出对话框，输入生成数量（1-100）
3. 选择是否使用当前配置
4. 点击"生成"，显示密码列表
5. 提供"全部复制"和"导出文件"选项

#### 导出格式
- **TXT 格式**：每行一个密码
- **CSV 格式**：包含序号、密码、强度、生成时间

### 5.5 密码短语生成器

#### 功能描述
生成基于随机单词的密码短语，更易记忆且安全性高（如：correct-horse-battery-staple）。

#### 配置选项
- 单词数量（3-8 个）
- 单词语言（英文/中文）
- 分隔符（-, _, 空格, 数字）
- 首字母大写
- 添加随机数字/符号

#### 词库要求
- 英文词库：5000+ 常用单词
- 中文词库：3000+ 常用词语
- 排除敏感词汇
- 优先选择短词（3-6 个字母/字）

## 7. 技术要求

### 7.1 前端技术栈
- **框架**：React 18+ / Vue 3+ / Next.js
- **语言**：TypeScript
- **样式**：Tailwind CSS / CSS Modules
- **状态管理**：Zustand / Pinia
- **构建工具**：Vite / Next.js
- **图标库**：Lucide Icons / Heroicons

### 7.2 安全要求
- 使用 Web Crypto API 生成随机数
- 所有密码生成逻辑在客户端执行
- 不向服务器发送任何密码数据
- 本地存储使用加密（历史记录）
- 添加 Content Security Policy (CSP)
- HTTPS 强制访问

### 7.3 性能要求
- 首屏加载时间 < 2s
- 密码生成响应时间 < 100ms
- Lighthouse 性能评分 > 90
- 支持 PWA（离线可用）

### 7.4 兼容性要求
- 现代浏览器最新两个版本（Chrome, Firefox, Safari, Edge）
- 移动端浏览器（iOS Safari 14+, Chrome Android）
- 不支持 IE 浏览器

### 7.5 SEO 要求
- 服务端渲染（SSR）或静态生成（SSG）
- 合理的 meta 标签
- 结构化数据（Schema.org）
- sitemap.xml 和 robots.txt
- Open Graph 和 Twitter Card

## 14. 附录

### 14.1 常见密码策略要求
- 银行：至少 8 位，包含大小写+数字+符号
- 邮箱：至少 10 位，包含大小写+数字
- WiFi：8-63 位，ASCII 字符
- Windows：至少 8 位，复杂度要求
- Linux：8-16 位，建议包含多种字符

### 14.2 密码熵计算公式
```
熵 = log₂(字符集大小^密码长度)

例如：
- 8位纯数字：log₂(10^8) ≈ 26.6 bits
- 12位大小写+数字+符号：log₂(94^12) ≈ 78.8 bits
- 16位大小写+数字+符号：log₂(94^16) ≈ 105 bits
```
